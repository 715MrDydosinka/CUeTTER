
CUeTTER - ffmpeg wrapper for splitting audio by CUE sheets

SYNOPSIS
    cuetter <cue-file> [OPTIONS] [-- FFMPEG_ARGS]

DESCRIPTION
    A convenient wrapper around ffmpeg that splits audio files according to
    CUE sheet timing information. All heavy lifting is done by ffmpeg - this
    tool simply generates and executes the appropriate ffmpeg commands.

REQUIRED
    -i, --input <file>        Input audio file (any format supported by ffmpeg)

OUTPUT OPTIONS
    -o, --output <dir>        Output directory for split tracks
                              (default: ./split_output)
    -f, --format <format>     Output container/codec (flac, mp3, wav, ogg, etc.)
                              Passed directly to ffmpeg as -f {format}

ENCODING OPTIONS
    These options are passed directly to ffmpeg with appropriate mapping:

    -b, --bitrate <rate>      Audio bitrate (e.g., 320k, 192k, V0)
                              Maps to: -b:a {rate}
    -r, --sample-rate <hz>    Sample rate in Hz (e.g., 44100, 48000)
                              Maps to: -ar {hz}
    -c, --channels <n>        Number of channels (1=mono, 2=stereo)
                              Maps to: -ac {n}

COVER ART
    -a, --album-cover <file>  Embed album cover image
                              Maps to: -i {file} -map 1 -c:v copy
                              (requires ffmpeg with image2 support)

METADATA
    --embed-metadata           Copy metadata from input/CUE to output files
                              Maps to: -map_metadata 0 -id3v2_version 3
    --embed-cue                Embed original CUE sheet in tracks metadata
                              Maps to: -metadata CUE="{cue_content}"

GAP HANDLING
    --pregap <ms>              Add silence at beginning of each track (ms)
                              Implemented via ffmpeg's adelay filter
    --postgap <ms>             Add silence at end of each track (ms)
                              Implemented via ffmpeg's apad filter

CONTROL FLAGS
    -w, --overwrite            Overwrite output files without asking
                              Maps to: -y (auto-confirm ffmpeg overwrites)
    -n, --dry-run              Show ffmpeg commands without executing them
    -v, --verbose              Show ffmpeg command lines and progress
    --version                  Display version information
    --help                     Display this help message

FFMPEG PASSTHROUGH
    -- [args]                  All arguments after -- are passed directly to
                              every ffmpeg invocation (must be last option)

EXAMPLES
    Basic split:
        cuetter album.cue -o ./tracks

    Split to high-quality MP3:
        cuetter album.cue -f mp3 -b 320k -o ./mp3s

    Split with cover art and metadata:
        cuetter album.cue -a cover.jpg --embed-metadata

    Preview ffmpeg commands (dry run):
        cuetter album.cue -n -v

    Pass additional ffmpeg filters:
        cuetter album.cue -o ./tracks -- -af "loudnorm"

DEPENDENCIES
    ffmpeg - Must be installed and accessible in PATH or 
        provided via enviroment variable 'ffmpeg='
    Supported on: Linux, macOS, Windows (WSL/Cygwin)
